Inverse analysis, shipped: ask the engine which sets fit

Every other tool answers "what is this set?". search_identities answers the inverse — "which sets satisfy these constraints?" — by exact, exhaustive enumeration over all 4096 pitch-class-set identities. The marquee agent-facing capability, built on the ruleset engine's own predicate, pointed the other way.

The query model

Blind-agent safe: strict, total validation

A malformed query returns every problem at once — the ruleset-validator contract — so an agent repairs in one round trip instead of whack-a-mole.

# search_identities({"cardnality": 7, "ic1": {"bogus": 1}, "contains": [13]}) ✗ 'cardnality': unknown field (known: cardinality, contained_in, contains, ic1…ic6, is_achiral, no_consecutive_semitones, rotational_period) ✗ ic1: operator object must have exactly one of ['in', 'gte', 'lte'] ✗ contains[0]: pitch class must be an int in 0..11, got 13

Decision 12 — containment granularity follows universe granularity

In the default set-class universe a shape and its mirror are the same class, so contains folds inversions — otherwise a rooted test against Rahn's arbitrarily-handed prime form would answer differently for a chiral set than for its mirror. In expand_transpositions (rooted masks) it's strict: [0,4,7] is the major triad, not the minor. Corollary: signed chirality is deliberately not a set-class field — only is_achiral, a true T/I-invariant, is. Handedness-aware search waits for search_voicings.
