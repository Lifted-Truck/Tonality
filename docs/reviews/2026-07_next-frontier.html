Four moves

Each is self-contained enough to ship as its own arc. Cost is rough effort; leverage is toward the North Star (agent-facing dataset foundation).

Recommended Phase 4 · constraint search

Inverse analysis: query the identity universe

search_identities() / search_voicings() — exact queries over the 4096-identity space and voicing spaces. "All 7-note scales containing this tetrachord with no consecutive semitones"; "all Cmaj9 voicings under 19 semitones with no ♭9 on the bass." The marquee agent tool, and the one new capability class vs. polish.

Why nowSubstrate (3.5a tables) is already built; substrate accrues nothing new by waiting
Costfocused greenfield, well-fenced
ServesNorth Star ★ generative-side

The sequencing nuance behind the recommendation

The ROADMAP's own note: "a search constraint and a checkable rule are the same predicate pointed in different directions — build the constraint vocabulary once." Phase 4.6's evaluator already is that predicate layer. So the smart version of the recommended move isn't a standalone search engine — it's constraint search built on the existing rules/ predicate vocabulary, which delivers the marquee tool and unifies search + rules under one predicate contract. That's the plan I'd scope if you pick A.
