Tonality · gap B · design brief · 2026-07-07

The harmony/progression rule family — the one blocker between today and a trance-harmony ruleset

A fourth atom family for the ruleset DSL, alongside voice-motion, melody, and rhythm. It makes chord-succession rules — "V resolves to I", "no ♭VII in this style", "prefer descending-fifth motion", "forbid retrogression" — expressible for the first time. It's the harmony row of every style profile, trance included.

No new theory — a new vocabulary over shipped analysis. Roman numerals + roles come from theory/functions.py; the dozen succession tags from tag_transition; cadence membership from detect_cadences. gap B is the field/enum registry + the chord-stream item builder + the same evaluator/induction wiring the other three families already use. The analysis exists; only the rule vocabulary is missing.

01The item — a chord in context

Each family quantifies over its natural item (a voice-pair transition, one note). Harmony's item is a chord in a key, carrying its motion to the next — so one item type supports both per-chord rules and adjacent-pair (transition) rules, exactly as voice_motion items already carry from/to.

03The fork — your call

FORK · input form

Explicit chord stream, or auto-derive from a Sequence?

The other three families derive their atoms from a raw Sequence (notes). Harmony could too — but chord segmentation from raw notes is a separate, noisier problem. The alternative: the caller hands over the named progression + key, the exact form detect_cadences already takes — [(root_pc, quality), …] in a key.

Recommend: explicit chord stream for slice 1. Clean, deterministic, and it isolates the deliverable — the vocabulary and its evaluation — from harmonic-segmentation quality. Auto-derive-from-Sequence (reusing the shipped segmentation + dataset records) is slice 2, added behind the same family once the vocabulary is proven. This matches how detect_cadences already works, so consumers have one consistent harmonic input form.

04Honest limits (what gap B still won't do)

gap B is necessary but not sufficient for two marquee asks — worth stating plainly:
  • Wend's "authentic cadence within 4 bars of a section end" is only half unblocked: gap B supplies the cadence field, but "within 4 bars of a section end" needs phrase/global scope — the same deferred gap the Fux ruleset just surfaced. gap B + phrase-scope together finish it.
  • "No more than two consecutive borrowed chords" needs run-length (gap C). gap B gives the field, gap C gives the sequential quantifier.
  • Major/minor only, inherited from theory/functions.py — modal keys return no roles (the engine won't claim function it can't ground). Trance's natural-minor / Aeolian idiom mostly fits minor; true modal styles wait on a functions expansion.
None of these block the trance harmony tendencies (which are succession + diatonicity + cadence-type rules) — they block the positional and run refinements.

Slice 1 plan: the harmony family (field/enum registry + the chord-in-context item builder over an explicit [(root,quality)]+key stream), wired into evaluate and induce_ruleset (both inherit it), the manifest, and a trance-flavored example ruleset in the library. Full ship-first field set (all listed above already exist). Additive golden, port pin untouched, the usual test bar. Both named consumers — A9 Wend (R4) and A10 wont (harmony scope) — unblock (partially, per the limits above).
